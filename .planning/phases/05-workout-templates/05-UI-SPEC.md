## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding between label and icon |
| sm | 8px | Compact element spacing, badge internal padding |
| md | 16px | Default element spacing, card internal padding |
| lg | 24px | Section padding, template card vertical padding |
| xl | 32px | Layout gaps between sections |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level top/bottom padding |

Exceptions:
- Minimum touch target: 44px height/width for all interactive controls — swipe action buttons, "Iniciar" button, exercise picker rows, set count steppers.
- Template card minimum height: 80px to accommodate name + meta row (exercise count + muscle tags) without crowding.
- Exercise picker row: 52px height — name at 16px with 18px muscle group label below, 8px vertical padding each side.

Source: Phase 3 UI-SPEC §Spacing Scale, REQUIREMENTS.md §UI-03, CONTEXT.md D-10 (card-based list).

## Templates Tab Architecture

**Route:** `/templates`

**Page layout:**
- Header: "Templates" in Heading/20px/600, right side: "Novo Template" button (ghost variant, `LayoutTemplate` icon + label)
- Body: scrollable `TemplateCard` list, vertical gap 12px (between md 16px and sm 8px — nearest even step allowed: use 12px as `gap-3`)
- Empty state: centered, no cards present

**Sort order:** Most recently used first (descending `lastUsed` timestamp); fallback to most recently created.

Source: CONTEXT.md D-12 (dedicated Templates tab), D-10 (card-based list). Sort order: Claude's discretion.

## Save-as-Template Flow (Post-Workout)

**Trigger:** "Salvar como Template" action in `SessionDetail.tsx` post-workout summary.

**Interaction:**
1. User taps "Salvar como Template" (ghost button in post-workout summary, `BookmarkPlus` icon 16px prefix).
2. shadcn `Dialog` opens (centered modal, not bottom sheet — avoids nav occlusion on session screen).
3. Dialog content: title "Salvar como Template" (Heading 20px/600) + name `Input` pre-filled with auto-generated name (e.g. "Treino Seg, 22 abr") + "Salvar" primary button + "Cancelar" ghost button.
4. On confirm: template saved, Dialog closes, success Toast: "Template salvo com sucesso."

**Auto-name format:** `"Treino {weekday abbr}, {day} {month abbr}"` (Portuguese, e.g. "Treino Seg, 22 abr") — editable before saving.

Source: CONTEXT.md D-04, D-05 (save-from-workout path). Auto-name strategy: Claude's discretion.

## Interaction Contract

### Start-from-Template Flow
1. User taps "Novo Treino" on home screen OR navigates to Templates tab.
2. Template list shown. Taps "Iniciar" on a `TemplateCard`.
3. `TemplatePreviewSheet` opens — user sees exercise list, can add/remove/reorder in edit mode.
4. Taps "Iniciar Treino" to confirm.
5. Session starts with inputs pre-filled from template (weights, reps, rest config).

### Template Creation (From Scratch)
1. User taps "Novo Template" on Templates tab.
2. `TemplateBuilderSheet` opens.
3. User enters name, adds exercises from `ExercisePicker`, configures set count per exercise.
4. Taps "Criar Template" → saves to Dexie templates table → Sheet closes → new card appears in list.

### Template Deletion
1. User swipes left on `TemplateCard` to reveal red delete zone, OR opens overflow menu and taps "Excluir".
2. Confirmation: inline confirm row replaces swipe zone — "Tem certeza?" with "Excluir" (destructive, `#EF4444`) and "Cancelar" buttons. No separate Dialog.
3. On confirm: card removes from list with fade-out animation (150ms opacity 1→0 + height collapse 200ms).

### Two-Tap Rule Verification (D-07)
- Home → Templates tab: 1 tap (bottom nav)
- Templates → start session: 2 taps (Iniciar → Iniciar Treino confirm)
- All major template actions reachable within 2 taps. Satisfied.

## Animations

| Animation | Trigger | Spec |
|-----------|---------|------|
| Template card entrance | Page load / new template saved | stagger 30ms per card, each: opacity 0→1, translateY 8px→0, 180ms ease-out |
| Card delete | Confirm delete tapped | opacity 1→0 150ms + height collapse 200ms ease-in |
| Swipe reveal | Swipe left gesture | CSS `transform: translateX` following finger, snap to 80px on release |
| Sheet entrance | Any bottom sheet opens | translateY 100%→0, 250ms cubic-bezier(0.32, 0.72, 0, 1) — matches Phase 3 Dialog pattern |
| Toast appearance | Template saved | Phase 3 Toast animation — no change |

`prefers-reduced-motion`: all transitions collapse to `0.01ms` — existing rule in index.css handles this.
