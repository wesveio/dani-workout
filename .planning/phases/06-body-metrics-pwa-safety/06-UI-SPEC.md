| Accent (10%) | #FF3D3D | Reserved — see below |
| Accent Secondary | #FF8C00 | Reserved — see below |
| Muted | #888888 | Placeholder text, secondary labels, disabled states, chart gridlines |

| Destructive | #EF4444 | Delete entry confirmation button only |
| Foreground | #F5F5F3 | All body text, headings, input values |

Accent (#FF3D3D) reserved for:
- Primary CTA button ("Salvar Peso", "Salvar Medidas") fill
- Active bottom nav tab indicator dot or underline
- Chart line for bodyweight trend (primary line)
- Storage warning threshold indicator (when >80% quota used)

Accent Secondary (#FF8C00) reserved for:
- Secondary chart lines on measurements chart (waist line)
- iOS install banner background tint or left border accent stripe

Do not apply accent to: input focus rings (use neutral #2A2A2A at 60% opacity), disabled buttons, informational text.

## Layout Contracts

### Bottom Navigation

- Add "Corpo" tab to existing bottom nav (Layout.tsx)
- Icon: Lucide `Scale` or `Activity` — use `Scale` (weight scale, most semantically clear)
- Label: "Corpo"
- Tab position: 4th tab (between Templates and any existing 5th, or as 5th tab — match existing nav structure)
- Active state: accent dot (#FF3D3D) + foreground label
- Inactive state: muted (#888888) icon + label

### Body Metrics Screen Layout

Structure (top to bottom, full mobile viewport):
1. Screen header: "Corpo" title (28px Display) + "Registrar" button (accent)
2. Chart section: Bodyweight trend line chart with period tabs — card background (#1A1A1A), 200px chart height
3. Measurements chart: Collapsible section — collapsed by default if no measurement data, 160px chart height
4. Log history: Scrollable list of past entries (newest first) — each entry is a Card with date, weight, optional measurement pills, edit/delete actions
5. Photo gallery: "Fotos de Progresso" section header + photo grid

### Quick-Log Sheet (bottom sheet / half-screen)

- Triggered by "Registrar" FAB or header button
- Background: #1A1A1A surface, rounded top corners (border-radius 24px per tailwind xl config)
- Fields in order:
  1. Date selector (defaults to today, text button showing date, tap to change)
  2. Peso (kg) — large input, 28px numeric value, autofocus on open
  3. Collapsible "Medidas (opcional)" section with: Cintura, Quadril, Peitoral, Braços — all in cm, 2-column grid
  4. Notas — Textarea, optional, 2 rows
- Action row: "Cancelar" (ghost) + "Salvar" (accent fill, full-width)

### Photo Gallery Grid

- 3-column grid, gap: 8px (sm token)
- Thumbnail: 80px × 80px minimum, object-cover, rounded-lg (12px)
- Date badge: overlaid bottom-left, 12px text, semi-transparent dark background
- Tap: opens Dialog with full-size image + date + delete option
- Add photo: first grid cell is an "+ Adicionar" placeholder card (border-dashed, muted color)

### iOS Install Banner (UI-04)

- Placement: fixed bottom, above the bottom nav bar
- Height: 56px
- Background: #1A1A1A with 2px left border in #FF8C00 (accent secondary)
- Content: Lucide `Share2` icon + "Adicione ao início" text (14px Label) + "Dispensar" ghost button
- Timing: Show after 3 seconds on first visit if `window.navigator.standalone !== true` and user is on iOS Safari
- Dismissal: tap "Dispensar" → hide banner, store dismissed flag in localStorage, do not show again for 7 days
- Re-prompt: show again after 7 days if user returns without having installed

## Interaction Contracts

### Quick-log form
- Peso input: numeric keyboard (`inputMode="decimal"`), autofocused on sheet open
- Saving an entry with only weight (no measurements): valid — measurements are all optional
- Save success: dismiss sheet + show Toast "Peso registrado!" (2 seconds)
- Validation: peso must be > 0 and < 500 (reasonable range), show inline error "Peso inválido" below field if out of range

### Entry edit/delete
- Tap entry card → opens quick-log sheet pre-populated with entry values + "Editar Registro" heading
- Delete: swipe-to-reveal delete button OR long-press on entry card → reveals delete icon — then confirmation Dialog before permanent removal
- After delete: entry removed from list with fade-out animation (200ms), Toast "Registro excluído"

### Photo actions
- Tap "+ Adicionar" → `<input type="file" accept="image/*" capture="environment">` triggers camera/gallery sheet
- Compress to ~200KB before storing (browser-side Canvas resize)
- After compression, if still > 1MB, reject with error toast
- Tap photo thumbnail → Dialog with full-size photo + date + "Excluir Foto" button
- Storage indicator: small text below gallery header "X.X MB usados" in muted color, shown always when photos exist

### Charts
- Bodyweight chart: smooth line (Recharts `type="monotone"`), dot on last data point only, no dot on intermediate points
- Period tabs: switch chart data range without animation flash (data update only)
- Measurements chart: each body part is a separate Line, color-coded:
  - Cintura: #FF8C00 (accent secondary)
  - Quadril: #4EFF74 (legacy green, or any distinct color not accent red)
  - Peitoral: #4495FF (legacy blue)
  - Braços: #A78BFA (purple — introduce as 5th neutral accent for charts only)
- Legend: displayed below chart, tap to toggle line visibility
- Empty chart (no data): show empty state text centered in chart height container, no axes rendered

### iOS install banner
- Only show on iOS Safari (detect via `navigator.userAgent` containing "iPhone" or "iPad" + Safari, not Chrome or Firefox on iOS)
- Only show if `window.navigator.standalone !== true` (app not already installed)
- Do not show on desktop or Android (Android has native beforeinstallprompt)
- Slide-up animation from below nav bar on appear (200ms ease-out)
- Slide-down on dismiss (150ms ease-in)
