## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding between label and icon |
| sm | 8px | Compact element spacing, badge internal padding |
| md | 16px | Default element spacing, card internal padding |
| lg | 24px | Section padding |
| xl | 32px | Layout gaps between sections |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level top/bottom padding |

Exceptions:
- Minimum touch target: 44px height/width for all interactive controls (set complete button, previous-value copy trigger, chart metric tabs).
- Inline set row: minimum 48px height to accommodate weight + reps inputs plus stepper buttons on mobile.
- Chart container: 256px fixed height (h-64) — existing convention from ExerciseHistory.tsx.

Source: Phase 3 UI-SPEC §Spacing Scale, REQUIREMENTS.md §UI-03.

1. Primary action buttons (set complete "Marcar feito" / "Feito" state)

- Volume area: stroke `#FF8C00` (accentSecondary), fill gradient from `rgba(255,140,0,0.4)` to `rgba(255,140,0,0.05)` — replaces existing hardcoded green `#18D02E` in ExerciseHistory.tsx.

- Chart axis text: muted `#888888`.

**Accent is NOT used for:** general text, borders, backgrounds, chart fills on inactive states, icon fills on non-active controls, the previous-session ghost values.

## Inline Set Row Design Contract (LOG-01, LOG-02, LOG-03)

### Layout

Single-row grid (mobile-first):

```
[Série N] [Weight input + steppers] [Reps input + steppers] [RIR slider] [Complete button]
```

On mobile (< sm breakpoint): weight and reps stack vertically, complete button full-width below.
On sm+: all columns inline as currently implemented in SessionDetail.tsx grid.

### Previous Session Ghost Values (LOG-02)

- Rendered as a `<div>` immediately below each input field.
- Style: `text-[12px] text-muted leading-none mt-1` (presentational exception — 12px muted).
- Content: `"Ant: {value}"` — e.g. `"Ant: 60 kg"` for weight, `"Ant: 8"` for reps.
- Source: `lastLogsByExercise.get(exerciseId)?.sets[setIndex]` — already computed in SessionDetail.
- Shown only when a previous log exists for this exercise. Hidden (not rendered) when no history.
- aria-label on ghost: `"Valor anterior: {value}"`.

### Auto-Advance After Set Complete (LOG-03)

- On complete toggle: focus advances to the weight input of the NEXT incomplete set in the same exercise.
- If no next set in same exercise: focus advances to weight input of first incomplete set in the next exercise card.
- If all sets complete: focus stays on complete button; scroll to session footer.
- Implementation: `useEffect` watching `set.completed` transition → `document.getElementById('set-input-{exerciseId}-{nextIdx}').focus()`.
- Input IDs must follow pattern: `set-input-{exerciseId}-{absoluteIndex}` for focus targeting.

### PR Detection Badge (LOG-04)

- Trigger: on set save/complete, compare `set.weight` against `bestWeight` from all historical logs for this exercise.
- A PR is: current `set.weight > bestWeight` (strictly greater — ties do not trigger).
- `PrBadge` appears inline to the right of the set row complete button.
- Animation: `celebration-pop 0.6s ease-out` on mount (existing keyframe).
- Badge content: `<Trophy className="h-3.5 w-3.5 mr-1" /> PR!` — 11px/600/uppercase.
- Badge persists for the session duration (does not auto-dismiss).
- aria-live: `"polite"` on the badge container so screen readers announce PR without interrupting flow.

### Screen Header

```

             [Focus label · rest time — 14px/400 muted]
```

Back button: secondary variant, 44px touch target, `ArrowLeft` icon — existing pattern, no change.

### Metric Tab Bar (HIST-02, HIST-03)

Three pill tabs above chart:

| Label | Metric | Series color |
|-------|--------|-------------|
| Volume | total kg × reps per session | `#FF8C00` area |
| Carga | top weight per session | `#FF3D3D` area |
| 1RM Est. | Epley formula: `weight × (1 + reps/30)` | `#FF3D3D` dashed line, no fill |

- Active tab: accent `#FF3D3D` text + underline indicator.
- Inactive tab: muted `#888888`.
- Tab min-height: 44px.
- Switching tabs swaps the AreaChart series without page transition.

### Chart

- Container: 256px height (h-64) — matches existing ExerciseHistory convention.
- X-axis: session dates formatted `"D MMM"` — existing dayjs format.
- Y-axis: auto-scale, no axis line, no tick lines — existing convention.
- Tooltip: dark theme — background `#1A1A1A`, border `#2A2A2A`, text `#F5F5F3`.
- Recharts `ResponsiveContainer` width 100%.

### History Log List (HIST-01)

Each session entry row:

```
[Date · Session type · Week]          [Volume badge]
[Set 1: 60 kg × 8 @ RIR 2]  [Set 2: ...]
[Notes if any]

- Row background: secondary `#1A1A1A`, border neutral `#2A2A2A`, `rounded-xl`.
- Set chips: `rounded-lg border border-neutral/10 bg-card px-2 py-2` — existing convention.
- Completed set chip: accent `#FF3D3D` left-border `2px solid` to distinguish completed from uncompleted.
- PR sets: `Trophy` icon `#FF3D3D` inline in chip.

### PR Record Summary Row (HIST-01 extension)

```

- Background: neutral `#2A2A2A`, `rounded-xl`, padding md (16px).
- Icon size: 16px, accent color `#FF3D3D`.
- Values: 16px/600 foreground.
- Labels: 12px/400 muted.

## Interaction Contract

### Set Logging Flow (LOG-01 through LOG-04)

1. User opens SessionDetail for a session.
2. Each exercise card shows set rows pre-populated from last session (LOG-02 ghost values shown below inputs).
3. User edits weight/reps in the inline inputs using keyboard or ±stepper buttons.
4. User taps "Marcar feito" — set transitions to `completed: true`.
5. Focus auto-advances to next incomplete set input (LOG-03).
6. If `set.weight > bestWeight`: `PrBadge` mounts with `celebration-pop` animation (LOG-04).
7. Rest timer starts automatically (LOG-05 — Phase 3 behavior, already implemented, unchanged).

### History Navigation Flow (HIST-01, HIST-02, HIST-03)

1. User navigates to Progress tab (Histórico, tab 3 in nav).
2. Exercise list shown — tapping exercise name navigates to `/history/{exerciseId}`.
3. ExerciseHistory screen loads with metric tab bar defaulting to "Volume".
4. User switches tabs to see different metrics — chart updates in place, no navigation.
5. User scrolls below chart to see full session history list.

### Two-Tap Rule Verification

- Dashboard → Histórico tab: 1 tap (nav).
- Histórico → ExerciseHistory detail: 1 tap (exercise row).
- SessionDetail → set complete + PR badge: 1 tap.
- All flows within 2-tap limit. No modal chains required.

|-----------|---------|------|
| `celebration-pop` | PR badge mounts | Existing keyframe — scale 0.9→1.0 + opacity 0→1, 600ms ease-out |
